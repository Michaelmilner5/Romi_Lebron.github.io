task_motor.py

Purpose

This file implements the motor-control task for one wheel using encoder feedback and PI velocity control.

Main Role

task_motor.py is responsible for low-level closed-loop wheel control. It regulates wheel motion, publishes displacement and voltage information, and supports logging.

Key Responsibilities

  • Update encoder position and velocity
  • Compute PI control effort from the velocity error
  • Send effort commands to the motor driver
  • Publish wheel displacement in meters for the observer
  • Publish applied motor voltage for the observer
  • Support logging queues for debugging and analysis

Why This File Matters

This file matters because it closes the loop around each motor. That is a major reason the robot is repeatable and controllable instead of behaving like an open-loop toy.
