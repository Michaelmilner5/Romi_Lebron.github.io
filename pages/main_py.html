main.py

Purpose

This file initializes the complete robot system and acts as the integration layer between all subsystems. It sets up hardware interfaces, creates shared variables, instantiates all tasks, and runs the cooperative scheduler.

Main Role

main.py connects all modules into one functioning robot. It determines how data flows between tasks and how frequently each task executes.

System Architecture Overview

The system is built on a cooperative multitasking framework. Each subsystem runs as a task and communicates through Shares. The scheduler repeatedly executes tasks based on priority and timing.

Hardware Initialization

View Hardware Setup
  • PWM Timer: 20 kHz timer used for motor control
  • Motor Drivers: Left and right motor control objects
  • Encoders: Quadrature encoders for wheel position and velocity
  • Line Sensor Array: 8-channel reflectance sensor system
  • IMU: BNO055 connected via I2C
  • I2C Bus: 400 kHz communication interface

Task Breakdown

View Task Descriptions
  • Motor Tasks: Closed-loop PI velocity control for each wheel
  • Line Follow Task: Computes steering corrections
  • IMU Task: Reads heading and yaw rate
  • Observer Task: Estimates system states
  • Bump Task: Handles collision safety
  • Start Button Task: Controls system enable
  • Course Task: Runs full obstacle course FSM
  • User Task: Handles serial commands
  • Garbage Task: Manages memory

Why This File Matters
