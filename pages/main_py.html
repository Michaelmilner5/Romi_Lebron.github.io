main.py

Purpose

This file initializes the full robot system. It sets up hardware, creates shared variables and queues, instantiates the task objects, and registers all tasks with the scheduler.

Main Role in the Project

The file acts as the integration layer for the full system. While individual tasks contain the behavior logic, main.py is what connects all of the hardware and software pieces together.

Key Responsibilities

  • Initialize motor driver objects
  • Initialize encoder objects
  • Initialize line sensor and IMU interfaces
  • Create Shares and Queues
  • Create task objects
  • Add tasks to the scheduler

Important Objects and Variables

  • motor objects – interface to left and right wheel actuation
  • encoder objects – provide wheel position and velocity feedback
  • line sensor object – provides reflectance-based line position data
  • IMU object – provides heading and yaw-rate measurements
  • Shares and Queues – allow communication between all running tasks

Why This File Matters

This file is critical because it transforms a collection of independent modules into one working robot. Without this file, the motor control, state estimation, line following, and event handling tasks would not operate as one integrated system.
